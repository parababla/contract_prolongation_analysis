# Анализ эффективности пролонгации договоров

## Описание проекта
Данный проект представляет собой аналитическое решение для отдела сопровождения клиентов. Основная цель — оценка работы аккаунт-менеджеров по продлению (пролонгации) договоров с клиентами. 

В рамках проекта реализован автоматизированный расчет ключевых метрик удержания выручки и формирование итогового отчета в формате Excel для принятия управленческих решений.

## Исходные данные
Для анализа использовались два набора данных:
1.  **Данные о пролонгациях (`prolongations.csv`)**: реестр проектов с указанием ID, месяца завершения договора и ответственного аккаунт-менеджера.
2.  **Финансовые показатели (`financial_data.csv`)**: помесячная динамика отгрузок по каждому проекту.

## Методология расчета коэффициентов
Система оценки базируется на двух коэффициентах:

### 1. Коэффициент пролонгации 1-го месяца (K1)
Показывает долю выручки, сохраненную сразу после истечения основного договора.
*   **Знаменатель:** Сумма отгрузки в последний месяц действия договора.
*   **Числитель:** Сумма отгрузки в первый месяц после завершения.

### 2. Коэффициент пролонгации 2-го месяца (K2)
Оценивает эффективность «дожима» и возврата клиентов, которые не продлили контракт в первый месяц.
*   **Знаменатель:** Сумма отгрузки последнего месяца для проектов, **не имевших** отгрузок в первый месяц пролонгации.
*   **Числитель:** Сумма отгрузки во второй месяц после завершения.

## Реализация и этапы работы
В ходе выполнения проекта был разработан Python-скрипт (Jupyter Notebook), выполняющий следующие задачи:

1.  **Предобработка и очистка:**
    *   Обработка специфических статусов: исключение проектов со статусами `стоп` и `end` (завершение до срока).
    *   Логика «в ноль»: если текущая отгрузка равна 0, система автоматически использует данные предыдущего периода для корректного расчета базы пролонгации.
    *   Группировка финансовых транзакций по ID проектов.
2.  **Расчет метрик:** Построчное вычисление целевых показателей и фактической выручки для каждого проекта.
3.  **Агрегация:** Сведение данных на уровне отдела, индивидуально по менеджерам и в помесячной динамике за 2023 год.
4.  **Формирование отчета:** Использование библиотеки `openpyxl` для генерации стилизованного Excel-файла.

## Результат
Итоговый аналитический отчет (`report.xlsx`) содержит три листа с настроенным форматированием (цветовое выделение, границы, объединение ячеек):
*   **Весь отдел**: сводные показатели эффективности всей команды по месяцам.
*   **Менеджеры за год**: годовые итоги работы каждого сотрудника.
*   **Менеджеры по месяцам**: детализированная матрица коэффициентов в разрезе каждого менеджера и периода.
